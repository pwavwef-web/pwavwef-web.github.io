1 — Overview & Motivation

The AZ Learner repository grew organically and accumulated several pages that no longer served the platform's purpose: personal portfolio pages built for individual team members, dead-end pages with no navigation entry, and files importing an inconsistent version of the Firebase SDK.

This audit was conducted to:

  • Remove pages that are personal-use only and unrelated to AZ Learner's student utility mission.
  • Standardise the Firebase JavaScript SDK to version 11.0.2 across the entire codebase.
  • Update CEO/admin quicklink panels to remove broken links to deleted pages.
  • Document all changes in this report for future reference.
No functional user-facing features were removed. All changes are backwards-compatible. Existing redirect stubs (e.g. resume.html → cv/) were left untouched.

3 — Firebase SDK Version Standardisation

Seven files were importing Firebase JavaScript SDK version 11.6.1 instead of the project-standard 11.0.2. Loading multiple SDK major-versions in the same browser session can cause subtle auth/Firestore state conflicts when users navigate between pages.

Import Change Applied
// BEFORE — incorrect version
import { initializeApp } from "https://www.gstatic.com/firebasejs/11.6.1/firebase-app.js";
// AFTER — standardised to project version
import { initializeApp } from "https://www.gstatic.com/firebasejs/11.0.2/firebase-app.js";

5 — Security Notes

Previous sessions identified and fixed an XSS vulnerability in sendSystemNotification() in both pwa/index.html and home.html. The fix replaced innerHTML assignment with textContent for user-controlled title and body fields. This fix is confirmed present and intact.

The formatBlogText() function correctly HTML-escapes input before applying markdown-like regex transforms, preventing injection through blog post content.

Deleting emmanuel.html also eliminates a page that stored a personal admin email address as a hardcoded JavaScript constant visible in the page source.

Outstanding recommendation: Review all hardcoded email constants across CEO and admin pages. Prefer storing sensitive role-check values in Firestore (e.g. users/{uid}.role) rather than as plain-text JS strings in page source.

7 — Recommended Next Steps
Add 404 redirect page
Create a custom 404.html that redirects visitors who followed old links to emmanuel.html etc. back to the homepage.
Move root-level HTML pages into subdirectories
Pages like ambassador.html, leaderboard.html should ideally live at ambassadors/portal.html, leaderboard/index.html. Leave redirect stubs at old URLs.
Audit duplicate pages
certificate.html (standalone) vs courses/cert.html (course certificates) serve different purposes — confirm both are still needed and add clear titles to distinguish them.
Move binary assets out of repo
Files like app.apk, appv2.0.apk, resume.pdf, resume2.pdf, learn-your-way.wav inflate repository size. Consider hosting these on Firebase Storage instead.
Upgrade Firebase SDK globally to latest stable
Once stable with 11.0.2, consider a planned upgrade to the latest Firebase SDK. Update all CDN imports uniformly, then test auth, Firestore, Storage, and Vertex AI integrations.
