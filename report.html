Why this matters

The AZ Learner codebase standardises on Firebase JS SDK 11.0.2. Mixing versions (11.0.2 and 11.6.1) in the same Firebase project can cause subtle module resolution conflicts, cache inconsistencies, and unpredictable behaviour when pages are loaded from a service-worker cache. Standardising the version across all pages ensures reliable, consistent Firebase behaviour site-wide.

5 Decision Rationale

Why each change was made and the guiding principles behind them.

Removing personal pages from a public product repo

Personal portfolio and CV pages have no place in a product repository. They confuse the codebase, may inadvertently expose personal information, and add maintenance overhead. The AZ Learner brand pages are what should exist here.

Standardising the Firebase SDK version

A mixed-version Firebase setup is a latent risk. The service worker caches module URLs keyed by version, so a page on 11.6.1 and another on 11.0.2 will load different bundles — potentially with conflicting auth state or Firestore semantics. Using 11.0.2 everywhere matches the codebase convention and the version all other pages already use.

Removing dead navigation links

Dead links in the CEO command panel cause 404 errors and are confusing. Removing the entries keeps the panel clean and accurate.
